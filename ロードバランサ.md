# ロードバランサ
複数のサーバにアクセスを振り分ける装置、または仕組みのこと
## 負荷分散
1台のサーバだけだと処理しきれないアクセスも複数台に分散することで単体サーバの負荷を軽減する
## 可用性向上
複数のサーバで運用していれば仮に1台落ちたとしても他のサーバでカバーできる
## メンテナンス性
メンテナンスしたいサーバをロードバランサから切り離して、アクセスが来ない状態で対応ができる
## L4（トランスポート層）
TCP/UDPレベルでの振り分け
HTTPの内容は見えない
## L7（アプリケーション層）
HTTPヘッダーやURLを見て振り分け可能
## アルゴリズム
| 方式      | 内容            |
| ------- | ------------- |
| ラウンドロビン | 順番に回す         |
| 最小接続数   | 接続数が少ないサーバへ   |
| 重み付き    | 性能の高いサーバへ多く振る |
| IPハッシュ  | 同じユーザーは同じサーバへ |
## Sticky Session
ログイン管理などのcookie情報をセッションに格納するためには同じサーバに送りつ続けないとならない
DBやRedisなどにセッション管理を任せることで回避できる
## healthcheck
一定間隔でロードバランサにぶら下がっているサーバの死活監視を行い、落ちているサーバへのアクセスを振り落とす
## SSL終端
HTTPSの復号をロードバランサで行うことが多い
ロードバランサから先はプライベートネットワークである構成がほとんどなので、暗号化する構成は少ないのではないか
