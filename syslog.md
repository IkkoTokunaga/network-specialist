# Syslog
ログメッセージを転送する業界標準の管理プロトコル
何かのイベントが発生したら、それを自身のバッファやディスクに保存し、同時にSyslogサーバへ転送する
転送はUDPのユニキャストで行われ、ポート番号は「514」
## Severity
ログメッセージの重要度を表す
| Severityレベル | 名称                | 説明                         |
| ----------- | ----------------- | -------------------------- |
| **0**       | **Emergency**     | システム使用不可。即時対応必須（全停止・致命障害）  |
| **1**       | **Alert**         | 早急な対応が必要な重大障害（サービス停止・重大影響） |
| **2**       | **Critical**      | 重大な異常状態。放置すると停止やデータ損失に発展   |
| **3**       | **Error**         | 機能エラーが発生。業務影響あり            |
| **4**       | **Warning**       | 異常兆候・しきい値超過。即時影響はない        |
| **5**       | **Notice**        | 注意喚起レベルの事象。状態変化の通知         |
| **6**       | **Informational** | 情報提供のみ（起動・停止ログなど）          |
| **7**       | **Debug**         | デバッグ用詳細ログ。運用上は通常不要         |
